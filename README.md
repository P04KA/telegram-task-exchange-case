# Telegram Mini App "Биржа заданий"

Публичный кейс из портфолио: биржа Telegram-заданий в формате `Telegram Mini App` с ролями исполнителя, заказчика и администратора, внутренним балансом, модерацией выполнений и ручными выплатами.

## Что это за продукт

Платформа решает простую, но коммерчески важную задачу: заказчик покупает доступ к размещению заданий и пополняет бюджет, исполнители берут задания в Telegram, выполняют действия и получают вознаграждение во внутренний кошелек.

В текущей версии поддержаны:

- задания на подписку на канал
- вступление в чат
- запуск ссылки или поста
- ручная и автоматическая проверка там, где это позволяет Telegram API
- hold награды на 48 часов после подтверждения
- ручные выплаты с операционной обработкой через админку и Telegram-оповещения

## Ключевые сценарии

### Исполнитель

1. Заходит в Mini App через Telegram-бота
2. Берет задание и сразу открывает нужный Telegram target
3. Возвращается и запускает проверку выполнения
4. Получает подтверждение и награду в hold на 48 часов
5. Оформляет заявку на вывод с телефоном и банком

### Заказчик

1. Активирует доступ к публикации заданий
2. Создает задание и публикует его
3. Следит за бюджетом, прогрессом и статусом
4. Управляет заданием через паузу, остановку и пополнение

### Админ

1. Проверяет спорные выполнения и заявки на выплаты
2. Начисляет баланс и управляет пользователями
3. Скрывает, останавливает и удаляет задания
4. Получает Telegram-алерты о новых payout request

## Архитектура

- `apps/api` - NestJS backend, auth, wallets, tasks, executions, moderation, payouts, bot layer
- `apps/mini-app` - React Mini App для исполнителя и заказчика
- `apps/admin` - отдельная web-админка для операций и модерации
- `PostgreSQL` - основная база данных
- `Redis + BullMQ` - очереди, TTL-резервы, отложенная разблокировка наград
- `Caddy` - production reverse proxy и HTTPS

## Инженерные решения

### Telegram auth

Mini App авторизуется по `Telegram.WebApp.initData` с серверной валидацией подписи.

### Автопроверка заданий

Для `join_channel` и `join_chat` используется `getChatMember`, поэтому автопроверка работает только если бот добавлен администратором в целевой канал или чат.

### Hold награды

Подтвержденная награда не становится доступной сразу: сначала она уходит в hold на 48 часов, после чего отдельная очередь разблокирует выплату.

### Выплаты

Пользователь оформляет payout request с номером телефона и банком. Админы видят заявку в админке и получают Telegram-уведомления, после чего связываются с пользователем и выполняют ручной перевод.

## Локальный запуск

1. Поднимите инфраструктуру:

```bash
docker compose up -d
```

2. Скопируйте `.env.example` в `.env`

3. Установите зависимости:

```bash
npm install
```

4. Заполните тестовые данные:

```bash
npm run seed
```

5. Запустите приложения:

```bash
npm run dev:api
npm run dev:mini
npm run dev:admin
```

## Production

PUBLIC_HOST=example.com ./scripts/deploy-to-server.sh
